作为一个普通的java程序员工作两年接触过和使用的技术

2016年8月22日

14年毕业，一下子就工作两年。

工作到现在感觉到了一个小瓶颈，最近技术上的提升比较少。

我所在的互联网公司工作大部分时间是在写业务逻辑，提升最大的阶段是刚工作的第一年，工作第二年基本是熟练和深入一些了解前一年来和大学自己学到的东西。

最近也在接触学习一些新的东西。

工作两年使用到到技术大体如下，开发的时候使用的基本都是一些比较普通的技术，没有什么分布式、集群之类高大上的关键字：

后台开发语言：Java

模板引擎：Velocity

前端技术：JS/JQuery、AngularJS、HTML／CSS／LESS、Grunt、Gulp

代码托管：SVN、GIT

1、Java应用场景

在公司开发使用的是内部封装的一个框架，简单轻巧。总得来说用java来实现后台的业务处理、五行一个的空指针判断和在鄙视完Hibernate后自己封装Dbutils造的轮子ORM工具类。

使用Lucene优化搜索，Lucene是大学的时候就接触学习的，毕业设计还用到了，当然只是简单地封装使用。有很多封装Lucene的完整框架，比如solr和ES。

SQL编写，剔除页面及其他复杂的逻辑，归根到底SQL最终主要写了两个查询：

（1）对象ID列表

（2）统计

一些优化经验：简单，经常Explain、是否有索引。

java程序员就是喜欢封装。

2、Velocity

页面主要使用Velocity作为数据模版引擎，使用最多的velocity-toolbox。toolbox好处有不少，坏处也很多，比如在VM模板上面有很多逻辑。

3、javascript

基本一个功能java代码和JS代码的比例是1:3或者更多，不管你信不信，反正我是信了；JQuery事件、AJAX处理和DOM操作；有用过一下AngularJS，不过用得不好。额，好吧，NG还有使用失败的经验。

最近公司前端使用Gulp对公司几个项目的前端资源进行统一的工程化整理，感觉非常的厉害，所以自己也花了不少时间去研究学习。

ES6很多特性都非常的喜欢，不过现在还只有在写NodeJS的时候用到。

4、HTML/CSS

主要写一些简单的、基本的样式布局，大体上是由专业前端来写的

5、缓存

缓存使用了J2Cache，因为一些基础的方法都是封装好了的

项目地址［Git@OSC］http://git.oschina.net/ld/J2Cache

6、代码托管

一开始组内只有几个人做开发，第一年在公司内部使用的还是SVN。但是后面人越来越多，业务越来越多就感觉有些使用上的问题了，所以在16年年初的时候全面切换到Git。因为公司
有自己的代码托管服务 git.oschina.net，所以就很方便地切换过来了。一开始使用Git的时候还遇到一些问题，但是一周之后就解放了，经过一段时间的实践再根据Git组的建议，现在使用
fork项目PR模式进行开发，非常方便。

其他

Node

最近在学NodeJS， 怎么说，很喜欢Node。

Mongo

看到自带聚合的时候我就开始膜拜了。

最后

正如标题写的那样，我是一个普通的程序员，喜欢看电影喜欢各种幻想，我掌握的技术没什么过人之处，当不懂的人说我厉害的时候，我其实是很心虚的，没啥的就是几个ifelse嘛，其实你也可以的。

当然，同行们但也不要太过于鄙视我，我跟大神们有很大的差距，但是跟你的差距只有3个月［哈，别打我，逃～］。
